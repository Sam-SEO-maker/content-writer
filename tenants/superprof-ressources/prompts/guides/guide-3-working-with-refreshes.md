## What is a Refresh?

---

A refresh is an update to an existing published article. Unlike new articles, refreshes require you to edit live content that readers can already see.

This creates two key differences:

1. The article must stay published throughout your work
2. Some elements (like certain links) were set up by SEO and shouldn't be changed

<summary><strong>Block References →</strong> Academic sources, studies or citations</summary>

- Quick command: `/block references` to insert.

Please type "block references" into the list of Gutenberg blocks available to you and only use this block.

This block should be used in all articles addressing serious topics that require a certain level of legitimacy (scientific articles, articles related to government information, etc.). You will add here any academic sources, government websites, or trustworthy sources of information to support the arguments you make in the articles.

> 🧠 If you're translating an article with sources from a blog into another language, try to check if the same sources are available in the language of your blog. Additionally, make sure the refreshed article uses the content chunking technique:

> 🧱 **Chunking**
>
> For the body text, the writer should use the chunking technique for structuring content into smaller, more focused sections (called "chunks") that AI systems can more easily process and extract information from.
>
> In practical terms, this means that each paragraph should be able to stand alone and make sense without requiring context from surrounding paragraphs. Chunked content uses:
>
> - Clear and descriptive headings (H1, H2 & H3) to separate different subtopics
> - Paragraphs consisting of a maximum of three or four sentences (this is a guideline but not a strict rule), and that focus on one main idea at a time.
> - Alternates two or three text paragraphs with other Gutenberg blocks (AI systems can more easily extract and cite specific points from an info block or a bulleted list, for example).

# Critical Rules

---

**1. Never take the article offline**

Work on the live version. Save frequently—as long as the saved version looks complete to readers, you can continue and save again later.

**2. Never change landing page link anchors**

Links pointing to `/lessons/` pages have anchor text set by the SEO team.

- ✅ You can edit the link to **other blog articles**. You should always change the anchor text and placement to make sure that they are according to our current guidelines.
- ❌ Never change anchor text for links to **landing pages**

If you're unsure whether a link goes to a landing page, check if the URL contains `/lessons/` (e.g., `/lessons/piano/london/` or; `/lessons/singing/canada/`). The URL will of course be in your local language.

## Using the correct editor

---

Some older articles use the classic editor instead of the block editor (also called Gutenberg). You need to convert these before making changes.

**How to recognise the classic editor:**
The editing area looks like a simple text box with a toolbar, not separate blocks.

**How to convert:**

1. Click "Text" tab (top right of editor), select all content, copy to a note as backup

    ![](https://prod-files-secure.s3.us-west-2.amazonaws.com/68e10e39-ef2a-4a05-9895-051e8245a633/0ee6b491-7b99-4a87-afeb-d9d87c28cc4a/Capture_decran_13.12.2024_a_10.52.03_AM.png)

2. Click "Switch to block editor" in the right-hand menu

    ![](https://prod-files-secure.s3.us-west-2.amazonaws.com/68e10e39-ef2a-4a05-9895-051e8245a633/70441343-2e94-415b-9d02-556a06e107c1/Capture_decran_13.12.2024_a_10.56.34_AM.png)

3. When prompted, select "Convert to blocks"

    ![](https://lh4.googleusercontent.com/s8liAOfi_bkZEbA3XpMCDFLFtV4_a8-n2-45Ca58BGjJcB_FZxd4hpWFbAs5B9fhLR4XtYkgUbIHECs8wwNquvLBJStUlafbIU4tGREpcA9Fw_V7Gytk9thHWM8VC0vqk3GCUuPuAmd7wSnmcgqhzw)

4. Check that all content converted correctly before continuing

> **IMPORTANT:**
>
> 1. If the conversion creates problems (broken formatting, missing content), let your contact person, SEO manager or SEO checker know. **Do not attempt to fix major conversion issues yourself.**
> 2. Do **not** switch back to the classical editor under any circumstances.

# Refresh Checklist

---

*Use this checklist in addition to the [Pre-Publication Checklist](/2dad6418695a802e93ffd07a863cfab7?pvs=25#2dad6418695a806a83f4d5a01c710ca4) when working on refreshes.*

**Before You Start**

- [ ] Confirm you're working in the block editor (Gutenberg)—if not, convert first (see above)
- [ ] Note which landing page links exist (you cannot change these)

**While Working**

- [ ] Article stays live—never switch to draft
- [ ] Save frequently (complete sentences/paragraphs only)

**Anchor Text Rules**

- [ ] Landing page links (`/lessons/` URLs): anchor text unchanged
- [ ] Blog article links: anchor text can be edited if needed

**Before Submitting**

- [ ] All mandatory blocks present (Key Takeaways, Info Box, Quote, Media+Text)
- [ ] Relevant tags added (Custom media if you added visuals)
- [ ] Preview checked on desktop and mobile
