# Content Writer

Autonomous **multi-tenant SEO refresh** agent: optimises existing content from
data signals (GSC + DataForSEO), preserving each tenant's editorial identity.
Data-driven decisions (audit → decision → generation → QC → internal linking).

Current tenants: `enseigna`, `superprof-ressources`. The registry is open
(`_shared/config/sites.json`): onboarding a market = `python3 content_writer.py
tenant init <id>` (skeleton `tenants/{id}/` + `sites.json` entry + sparse-checkout),
then fill in the editorial part. See [onboarding/](onboarding/README.md).

## Onboarding (new SEO Managers)

A market's SEO Manager (ES, UK, US, MX, ID, JP…) clones **only their tenant** via a
git sparse-checkout: the shared engine + their single `tenants/{id}/`, never the other
markets. Full walkthrough: **[onboarding/README.md](onboarding/README.md)**.

Recommended clone (sparse):

```bash
bash onboarding/scripts/setup_sparse.sh <tenant-id>   # e.g. es-es-ressources
```

## Installation (maintainer / full clone)

```bash
pip install -r requirements.txt
cp .env.example .env   # then fill in the credentials (DataForSEO, WP, GSC…)
```

## Usage

The CLI is `content_writer.py`. The up-to-date list of groups and commands is
auto-generated by Click:

```bash
python3 content_writer.py --help
python3 content_writer.py <group> --help    # e.g. refresh, batch, audit, tenant
python3 content_writer.py tenant list        # catalog of onboardable markets
python3 content_writer.py tenant init <id>   # onboard a tenant (skeleton + sparse)
```

Example: `python3 content_writer.py refresh <url> --blog enseigna`

## Golden Rule (invariant)

Never reduce a refreshed article's assets: `assets_after ≥ assets_before`
(images, tables, videos, internal **and** external links, including links to competitors).
